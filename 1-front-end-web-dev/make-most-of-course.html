2- How to get the most out of the course

Notes:

Use Cornell Note taking

  • Rather than coding along the same as the instructor, take key notes & review every 10 minutes or so
  • Pull out the main keywords/questions from the notes
  • Do a quick summary

Try to understand why the instructor is doing this

  • Why the code is important
  • Why it's being done this way

Then try to do the code yourself from memory

When you hit problems

  • One of the main parts of coding is hiting problems
  • Use Google & Stack Overflow to try to work it out yourself
  • Very likely someone else has hit this issue themselves 1st & you can find the solution online

Keywords/Questions:

  • Cornell Note taking
  • Why's the code important & being done?

Summary:

Don't just rush through the course, copying the code on the screen as won't learn a huge amount of it. Do Cornell note taking, after 10 minutes review it, understand why it's important & then do the code yourself. One of the main parts of coding is hitting problems. Research on Google and Stack Overflow.
