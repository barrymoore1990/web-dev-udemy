2 - Debugging & Syntax

Notes:

Debugging

  • Use chrome developer tools, console see if any issues
  • In elements, look at styles for the part you're looking at to see what's affecting it
  • Priority order - inline, internal then external

Anatomy of CSS Syntax

  • Syntax is grammar
  • Selector {property: value;}
  • Selector is the who - what you want to style. Property is the what - what do you want to style. Value is how you want to change it
  • Each property: value pair should be on their own line & in alphabetical order
  • MDN docs have index of all properties you can change

CSS Selectors

  • Add class to HTML elements e.g. img class="classattribute"
  • Comments are /* enter comment */
  • Break into sections on styles.css? Tag selectors at top then class selectors under
  • Don't need to use tag, just class
  • In style.css put . before class e.g. .bacon
  • Can use class many times
  • Any HTML element can have more than 1 class - class="bacon broccoli"
  • Used often and more than just e.g. h1

ID Selectors

  • Put ID selector section on styles.css under class selectors
  • Put # instead of . e.g. #bacon
  • Can only do one ID tag per page
  • HTML elements can only have 1 ID
  • Not used too often
  • Highest specificity

Pseudo Class

  • Some css classes are psuedo - they're different if you e.g. hover over the element
  • Hover most common
  • To use, you'd put after selector. e.g. img:hover

Keywords/Questions:

  • Chrome developer tools to find errors
  • Priority order - inline, internal, external. ID, class, tag
  • Syntax is grammar
  • class is . and id is #
  • Class is many, id is once

Summary:

Use chrome developer tools to find errors
