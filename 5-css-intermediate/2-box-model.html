2 - Box Model

Notes:

Box Model

  • If % it's of the viewport
  • Viewport is the screen your viewing the site on
  • Margin is a buffer zone outside the box
  • Padding is space inside the box
  • Lots of elements have default values & you have to change if you don't want the defaults
  • Border, margin and padding all increase size of box / space needed

Display Property

  • Display has 4 different values: block, inline, inline-block, none
  • Some (most/all?) elements have defaults
  • H1 & p are block which is the full width of the page (block out any other elements from sitting next to it)
  • Use span class to edit things inside a display (an inline element)
  • Spans don't do much unless paired with CSS
  • Inline elements only take up as much space as it needs to - common ones are span, a, img
  • If you set size for e.g. span width it doesn't change
  • Can change elements to different display values, e.g. p to inline
  • Inline-block gives you best of both - can change width and be on same line
  • Img's kind of inline-block by default
  • Display none gets rid of element
  • You might do none if e.g. got a quiz on your site and want to hide answer
  • Can also do with visibility: hidden. This keeps the elements space where it's supposed to be but hides content

Keywords/Questions:

  • % is of viewport
  • Lots of elements have default values
  • 4 display values, block, inline, inline-block & none
  • Elements have default display values but can be changed

Summary:

Margins, padding and borders affect size of boxes. All elements can be displayed inline, block, block-inline or none
