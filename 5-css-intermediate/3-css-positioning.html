3 - CSS Positioning

Notes:

CSS Positioning

  • Height is determined by the content, despite any CSS
  • Order on your screen comes from the code
  • Children sit on parents - away from screen and towards user
  • x & Y axis on screen but also a Z axis - determines what element will be displayed on another one

Position

  • Static, relative, absolute, fixed
  • Static - all HTML elements are static by default (go with HTML flow without changing)
  • Relative - relative to where it would be when static
  • Top, bottom, left and right coordinates to move elements
  • Relative positioning doesn't affect any other elements on the screen so can overlap
  • Can't use relative by itself, need to use coordinates
  • Left moves elements right, right moves elements left

Absolute

  • When you use coordinates e.g. right, it's now relative to its parent so works a lot differently than
  • Parent can be the entire page

Fixed

  • When set to fixed, the element doesn't move as you scroll

Centering

  • text-align: center. Works when we have an inline-block or block elements. Works when no width set
  • If width set, use margin:auto which will center the element (can do pixel margin on other part)

Keywords/Questions:

  • Children sit on parents (z-axis)
  • Static (normal), relative (to static position), absolute (relative to parent), fixed
  • text-align center or margin:auto to center

Summary:

Margins, padding and borders affect size of boxes. All elements can be displayed inline, block, block-inline or none
